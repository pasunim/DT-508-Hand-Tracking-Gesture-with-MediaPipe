# 🚀 Hand Tracking Space Game

เกม Space Shooter แนว Top-Down ที่ควบคุมด้วย **มือเปล่าผ่านเว็บแคม** โดยใช้ [MediaPipe Hands](https://developers.google.com/mediapipe) ตรวจจับตำแหน่งมือแบบเรียลไทม์ ไม่ต้องใช้เมาส์หรือคีย์บอร์ดในการเล่น

> Workshop ครั้งที่ 12 — วิชา DT-508 Machine Learning
> หัวข้อ: การสร้างสื่อเชิงโต้ตอบด้วย Hand Tracking และ Gesture ด้วย MediaPipe
> ผู้สอน: ผู้ช่วยศาสตราจารย์ บัญญพนต์ พูลสวัสดิ์

---

## 📸 ภาพรวมการทำงาน

```
┌──────────────┐    ┌───────────────────┐    ┌──────────────────┐
│   Webcam     │ →  │  MediaPipe Hands  │ →  │   Game Canvas    │
│ (ภาพวิดีโอ)    │    │ (ตรวจจับ landmark  │    │ (แปลงตำแหน่งมือ    │
│              │    │  ของมือ 21 จุด)     │    │  เป็นตำแหน่งยาน)   │
└──────────────┘    └───────────────────┘    └──────────────────┘
```

1. กล้องเว็บแคมส่งภาพทีละเฟรมให้ MediaPipe Hands ประมวลผล
2. MediaPipe คืนค่าตำแหน่ง landmark ของมือ (สูงสุด 2 ข้าง) เป็นค่าสัดส่วน 0–1
3. เกมนำตำแหน่ง **โคนนิ้วกลาง (Middle Finger MCP — landmark จุดที่ 9)** ของมือทั้งสองข้างมาหาจุดกึ่งกลาง เพื่อใช้เป็นตำแหน่งของยานอวกาศ
4. ระยะห่างระหว่างมือสองข้างใช้เป็น Gesture สั่งยิงกระสุน

## 🎮 วิธีเล่น

| ท่าทาง                                                                                      | ผลลัพธ์                                                                                            |
| ------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| ยกมือ**2 ข้าง** ให้กล้องเห็นพร้อมกัน                           | ยาน (สี่เหลี่ยม cyan) ขยับตามจุดกึ่งกลางระหว่างมือทั้งสอง |
| **กางมือ 2 ข้างออกจากกัน** จนระยะห่างเกิน 150 พิกเซล | ยิงกระสุน (มี cooldown 200 มิลลิวินาทีต่อนัด)                                 |
| หุบมือเข้าหากัน (ระยะ < 150)                                                   | หยุดยิง                                                                                            |

**เป้าหมาย:** ยิงศัตรูสีชมพูที่ตกลงมา เอาตัวรอดให้ถึง **วินาทีที่ 90** เพื่อเจอบอสสีแดง แล้วยิงบอสจนพลังชีวิต (HP 100) หมดเพื่อชนะ

- โดนศัตรูชน หรือโดนกระสุนบอส → **Game Over!**
- ยิงบอสจน HP หมด → **You Win!**

> ⚠️ ต้องเห็นมือ **ครบ 2 ข้าง** เท่านั้นยานถึงจะขยับ — มุมซ้ายบนของจอเกมมี Debug HUD แสดง `Hands: X/2` และ `Distance` ไว้เช็กสถานะการตรวจจับ (สีเขียว = จับมือได้ครบ)

## 📂 โครงสร้างโปรเจกต์

```
workshop_01/
├── index.html    # โครงหน้าเว็บ + โหลด MediaPipe จาก CDN
├── style.css     # จัด Layout กล้อง 30% / เกม 70%
├── game.js       # โค้ดเกมทั้งหมด (MediaPipe + Game Loop)
└── README.md     # ไฟล์นี้
```

### index.html

- โหลดไลบรารี 2 ตัวจาก CDN ของ jsDelivr:
  - `@mediapipe/camera_utils` — จัดการดึงภาพจากเว็บแคม
  - `@mediapipe/hands` — โมเดลตรวจจับมือ
- มี `<video id="input_video">` สำหรับแสดงภาพกล้อง และ `<canvas id="output_canvas">` สำหรับวาดเกม

### style.css

- ใช้ CSS Grid แบ่งจอเป็น 2 คอลัมน์: กล้อง `30%` / เกม `70%` (ปรับได้ที่ตัวแปร `--camera-width`, `--game-width` ใน `:root`)
- วิดีโอกล้องถูก `transform: scaleX(-1)` เพื่อกลับภาพเป็นกระจก
- Canvas เกมล็อกสัดส่วน 4:3

### game.js — แบ่งเป็น 4 ส่วนหลัก

| ส่วน                                           | บรรทัด (โดยประมาณ) | หน้าที่                                                                                                                        |
| -------------------------------------------------- | --------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| ตั้งค่า Canvas + ตัวแปรผู้เล่น | ต้นไฟล์                    | สร้าง context 2D, กำหนดขนาด canvas เท่าหน้าต่าง, ตัวแปร`player`                                     |
| ตั้งค่า MediaPipe                           | ถัดมา                        | `new Hands()`, `setOptions` (มือสูงสุด 2 ข้าง, ความมั่นใจ 0.5), `new Camera()` ที่ 640×480           |
| `onResults()`                                    | กลางไฟล์                  | รับผลตรวจจับทุกเฟรม — คำนวณตำแหน่งยานและระยะห่างมือ                                  |
| `gameLoop()`                                     | ท้ายไฟล์                  | วนด้วย`requestAnimationFrame` (~60 FPS) — วาดทุกอย่าง, ยิงกระสุน, ศัตรู, บอส, เช็กการชน |

## ⚙️ ค่าที่ปรับแต่งได้ (Game Balance)

| ตัวแปร                 | ค่าเริ่มต้น   | ความหมาย                                                                                                                       |
| ---------------------------- | ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------- |
| `shootThreshold`           | `150`                  | ระยะห่างมือ (px) ขั้นต่ำที่ทำให้ยิง — ลดลงถ้ารู้สึกว่าต้องกางมือกว้างไป |
| cooldown ยิง              | `200` ms               | หน่วงเวลาระหว่างกระสุนแต่ละนัด                                                                           |
| `player.size`              | `40`                   | ขนาดยานผู้เล่น                                                                                                           |
| อัตราเกิดศัตรู | `Math.random() < 0.02` | โอกาส 2% ต่อเฟรม (~1.2 ตัว/วินาที) — เพิ่มเลขให้ยากขึ้น                                        |
| ความเร็วศัตรู   | `3`                    | พิกเซลต่อเฟรม                                                                                                             |
| เวลาบอสเกิด       | `90` วินาที      | ลดลง (เช่น`10`) เพื่อทดสอบบอสเร็วๆ                                                                         |
| `boss.hp`                  | `100`                  | จำนวนนัดที่ต้องยิงบอส                                                                                             |
| อัตราบอสยิง       | `Math.random() < 0.05` | โอกาส 5% ต่อเฟรม ยิงกระจายครั้งละ 5 นัด                                                                 |
| `maxNumHands`              | `2`                    | จำนวนมือสูงสุดที่ MediaPipe ตรวจจับ                                                                            |
| `minDetectionConfidence`   | `0.5`                  | ความมั่นใจขั้นต่ำในการตรวจจับมือ (0–1)                                                                |

## 🚀 วิธีรันโปรเจกต์

**ต้องรันผ่าน HTTP server เท่านั้น** — เปิดไฟล์ตรงๆ แบบ `file://` จะใช้กล้องไม่ได้ เพราะเบราว์เซอร์อนุญาต `getUserMedia` เฉพาะ secure context (`https://` หรือ `localhost`)

### วิธีที่ 1: Live Server (VS Code) — แนะนำ

1. ติดตั้ง Extension **Live Server** ใน VS Code
2. คลิกขวาที่ `index.html` → **Open with Live Server**
3. เบราว์เซอร์จะเปิด `http://127.0.0.1:5500` ให้กด **Allow** เมื่อถูกขอสิทธิ์ใช้กล้อง

### วิธีที่ 2: Python

```bash
cd workshop_01
python3 -m http.server 8000
# เปิด http://localhost:8000
```

### วิธีที่ 3: Node.js

```bash
npx serve .
```

> 💡 ครั้งแรกที่เปิด อาจต้องรอ 2–3 วินาทีให้ MediaPipe โหลดไฟล์โมเดลจาก CDN (ต้องต่ออินเทอร์เน็ต)

## 🔧 การแก้ปัญหา (Troubleshooting)

| อาการ                                                | สาเหตุ / วิธีแก้                                                                                                                                                                                                         |
| --------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ภาพกล้องไม่ขึ้น                            | ยังไม่ได้กด Allow สิทธิ์กล้อง — ดูไอคอนกล้องที่ address bar / เช็กว่าเปิดผ่าน`localhost` ไม่ใช่ `file://` / กล้องถูกแอปอื่นใช้อยู่ (Zoom, Teams) |
| จอเกมดำสนิท ไม่มีตัวหนังสือ HUD | JavaScript error — กด`F12` เปิด Console ดูข้อความสีแดง มักเกิดจากโหลด CDN ไม่ได้ (เน็ตหลุด)                                                                                        |
| HUD ขึ้น`Hands: 0/2` ตลอด                       | มือไม่อยู่ในเฟรมกล้อง — ถอยห่างจากกล้อง, เพิ่มแสงสว่าง, อย่าให้มือหลุดขอบภาพ (กรอบกล้องแคบแค่ 30% ของจอ)                                    |
| HUD ขึ้น`Hands: 1/2` ยานไม่ขยับ           | เป็นไปตามเงื่อนไขเกม — ต้องยกมือ**2 ข้างพร้อมกัน**                                                                                                                                    |
| ยิงไม่ออก                                        | ดูค่า`Distance` บน HUD — ต้องกางมือจนเกิน 150                                                                                                                                                               |
| ยานกระตุก / หน่วง                           | ลด`modelComplexity` จาก `1` เป็น `0` ใน `hands.setOptions()` เพื่อใช้โมเดลเบาลง                                                                                                                  |
| ทิศทางซ้าย-ขวากลับด้าน               | ภาพกล้องถูก mirror ด้วย CSS แต่พิกัดเกมไม่ได้ mirror — ถ้าอยากให้ตรงกัน แก้ใน`onResults` เป็น `player.x = canvasElement.width - (x1 + x2) / 2;`                         |

## 🎨 ต่อยอด (Workshop เพิ่มเติมจากสไลด์)

**เปลี่ยนยานสี่เหลี่ยมเป็นรูปภาพจรวด:**

1. สร้างโฟลเดอร์ `images/` แล้วนำรูปมาใส่เป็น `images/space.webp`
2. ประกาศตัวแปรโหลดรูปที่ต้นไฟล์ `game.js`:
   ```js
   const playerImage = new Image();
   playerImage.src = 'images/space.webp';
   ```
3. แทนที่ส่วนวาดผู้เล่น (สี่เหลี่ยม cyan) ด้วย:
   ```js
   if (playerImage.complete && playerImage.naturalWidth > 0) {
       canvasCtx.drawImage(
           playerImage,
           player.x - player.size / 2,
           player.y - player.size / 2,
           player.size,
           player.size
       );
   }
   ```
4. ปรับ `player.size` จาก `40` เป็น `100` ให้รูปเห็นชัดขึ้น

ไอเดียต่อยอดอื่นๆ: ระบบคะแนน/ชีวิต, เสียงเอฟเฟกต์, ใช้ Gesture อื่น (เช่น กำมือ = โล่), แถบเลือด HP ของบอส, หน้าจอ Start/Restart แทน `alert()`
